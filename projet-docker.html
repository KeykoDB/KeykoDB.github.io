Projet Docker

Docker est un logiciel libre de conteneurisation. Il permet de lancer des applications dans un environnement isolé de la machine physique. Contrairement à la virtualisation qui permet de recréer une machine et ses composants dans un état dit « virtuel », Docker fonctionne sur le noyau et les composants de la machine physique. Ainsi, il est plus léger qu’une machine virtuelle et donc plus rapide mais il n’est pas possible de faire tourner un Docker Windows sur une machine hôte Linux, et inversement.

Une machine virtuelle « imite » intégralement un serveur. Dans un serveur virtualisé, chaque VM contient un système d’exploitation complet, avec ses pilotes, fichiers binaires ou bibliothèques, ainsi que l’application elle-même. Contrairement au concept de conteneurisation qui permet aux instances virtuelles de partager un système d’exploitation hôte unique, avec ses fichiers binaires, bibliothèques ou pilotes.

Objectif du Projet

L'objectif du projet était de réaliser une application pour une entreprise cliente (Tibco). X-SIRH est la future application de gestion des ressources humaines de l’entreprise. Elle reposera sur une base de données existante implantée de préférence sur un serveur Oracle.
